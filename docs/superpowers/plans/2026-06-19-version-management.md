# Version Management (semantic-release + npm OIDC) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add commit-driven automated releases (version bump + CHANGELOG + npm publish + GitHub Release) to `hostdoc` with zero stored secrets, triggered manually.

**Architecture:** semantic-release reads Conventional Commits to compute the next version, runs a 6-plugin pipeline (analyze → notes → changelog → npm → github → git), and publishes to npm via OIDC trusted publishing (no NPM_TOKEN). A manual `workflow_dispatch` GitHub Actions workflow runs it; the existing credential-free `ci.yml` is untouched.

**Tech Stack:** `semantic-release@^25`, `@semantic-release/changelog@^6`, `@semantic-release/git@^10` (the npm/github/commit-analyzer/release-notes-generator plugins ship bundled with semantic-release@25, which pins `@semantic-release/npm@^13.1.1`), GitHub Actions, npm OIDC trusted publishing.

## Global Constraints

- Release runner: **Node ≥ 22.14.0** and **npm CLI ≥ 11.5.1** (Node 22 bundles npm 10 → upgrade step mandatory). Source: https://docs.npmjs.com/trusted-publishers/
- **GitHub-hosted runner only** (`ubuntu-latest`); self-hosted runners do not support npm OIDC.
- **No NPM_TOKEN anywhere.** npm auth is OIDC (`id-token: write`); GitHub Release auth is the built-in `GITHUB_TOKEN`.
- `@semantic-release/npm` must be **≥ 13.1.0** (OIDC support) — satisfied transitively by `semantic-release@^25`.
- First release is **1.0.0** (semantic-release default; no special config).
- Trigger is **`workflow_dispatch` only** — never `push`.
- Commit-back message must contain **`[skip ci]`** so `ci.yml` does not re-run.
- The published npm tarball must contain **only `dist/`** (+ package.json/README/LICENSE), per `files: ["dist"]`. Build with `tsc` (`npm run build`), never tsgo.
- Do **not** modify `.github/workflows/ci.yml`.
- Commit types for this plan's own commits use `build:`/`ci:`/`docs:` (not `feat`/`fix`) — they are tooling changes.

### Task 2: Add the semantic-release configuration

**Files:**
- Create: `.releaserc.json`

**Interfaces:**
- Consumes: the six plugins verified in Task 1.
- Produces: a `main`-branch release config consumed by `npx semantic-release` in Task 3.

- [ ] **Step 1: Create `.releaserc.json`**

```json
{
  "branches": ["main"],
  "plugins": [
    "@semantic-release/commit-analyzer",
    "@semantic-release/release-notes-generator",
    "@semantic-release/changelog",
    "@semantic-release/npm",
    "@semantic-release/github",
    [
      "@semantic-release/git",
      {
        "assets": ["CHANGELOG.md", "package.json", "package-lock.json"],
        "message": "chore(release): ${nextRelease.version} [skip ci]\n\n${nextRelease.notes}"
      }
    ]
  ]
}
```

Notes: plugin order is the documented recipe — `changelog` writes `CHANGELOG.md`, `npm` updates `package.json` version + publishes, `github` creates the Release, `git` commits the assets back. The `[skip ci]` token keeps `ci.yml` from re-running on the commit-back.

Run:
```bash
node -e "const c=require('./.releaserc.json'); if(c.branches[0]!=='main'||c.plugins.length!==6) throw new Error('bad config'); console.log('config ok, plugins:', c.plugins.length)"
```
Expected: `config ok, plugins: 6`

- [ ] **Step 3: Commit**

```bash

### Task 3: Add the manual release workflow

**Files:**
- Create: `.github/workflows/release.yml`

**Interfaces:**
- Consumes: `.releaserc.json` (Task 2), `npm run build` (Task 1).
- Produces: a `workflow_dispatch` job with an optional `dry_run` boolean input.

- [ ] **Step 1: Create `.github/workflows/release.yml`**

```yaml
name: Release

on:
  workflow_dispatch:
    inputs:
      dry_run:
        description: "Dry run (compute next version + notes, publish nothing)"
        type: boolean
        default: false

permissions:
  contents: write        # GitHub Release, tag, and commit-back
  issues: write          # comment on released issues
  pull-requests: write   # comment on released PRs
  id-token: write        # npm OIDC trusted publishing + provenance

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0           # semantic-release needs full history + tags
      - uses: actions/setup-node@v4
        with:
          node-version: 22          # ≥ 22.14 (semantic-release engines + OIDC)
      - run: npm install -g npm@latest   # ≥ 11.5.1 for OIDC (Node 22 ships npm 10)
      - run: npm ci
      - run: npm run build
      - run: npx semantic-release ${{ inputs.dry_run && '--dry-run' || '' }}
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

Notes: no `NPM_TOKEN` — the npm plugin uses the OIDC token enabled by `id-token: write`. `secrets.GITHUB_TOKEN` is injected automatically by GitHub; no manual secret. The `dry_run` input lets you validate end-to-end (real OIDC + token context) without publishing.

- [ ] **Step 2: Verify the workflow is valid YAML**

Run (macOS ships Ruby with the YAML library):
```bash
ruby -ryaml -e "y=YAML.load_file('.github/workflows/release.yml'); raise 'no dispatch' unless y[true] && y[true]['workflow_dispatch']; raise 'no id-token' unless y['permissions']['id-token']=='write'; puts 'workflow ok'"
```
Expected: `workflow ok`. (Optional stronger check: `actionlint .github/workflows/release.yml` if installed.)

- [ ] **Step 3: Commit**

```bash
git add .github/workflows/release.yml
git commit -m "ci: add manual release workflow (semantic-release via npm OIDC)"
```

### Task 4: Document the one-time bootstrap and release procedure

**Files:**
- Create: `docs/RELEASING.md`

**Interfaces:**
- Consumes: the workflow (Task 3) and config (Task 2).
- Produces: operator-facing runbook (referenced by Task 5's go-live).

- [ ] **Step 1: Create `docs/RELEASING.md`**

```markdown
# Releasing hostdoc

Releases are automated with [semantic-release](https://semantic-release.gitbook.io/)
and published to npm via **OIDC trusted publishing** — there is no stored npm token.

## One-time bootstrap (first release only)

Trusted publishing is configured per npm package, and `hostdoc` does not exist on
npm yet. So the very first publish is manual, after which all releases are automated.

1. **Claim the package name with one manual publish** from your machine:
   ```bash
   npm login            # interactive
   npm run build        # emits dist/
   npm publish          # publishes the current package.json version
   ```
   (Alternatively use a short-lived npm automation token for this single publish.)
2. On **npmjs.com → the `hostdoc` package → Settings → Trusted Publishers**, add a
   GitHub Actions publisher: this repository + workflow file `release.yml`
   (+ environment if you use one). Save.
3. From now on, never publish manually — use the workflow below.

> If npm lets you register the trusted publisher before the first publish, you may
> skip the manual `npm publish` and run the workflow for 1.0.0 directly. Confirm the
> current flow in the npm docs: https://docs.npmjs.com/trusted-publishers/

## Cutting a release

1. Make sure the commits you want released are merged to `main` and follow
   Conventional Commits (`feat:` → minor, `fix:` → patch, `feat!:`/`BREAKING CHANGE:` → major).
2. **Dry run first** (recommended): GitHub → Actions → **Release** → *Run workflow* →
   `dry_run = true`. Read the log: it prints the next version and release notes,
   publishes nothing.
3. **Real release**: *Run workflow* with `dry_run = false`. semantic-release bumps
   the version, updates `CHANGELOG.md`, publishes to npm (with provenance), creates
   the GitHub Release, and commits `CHANGELOG.md`/`package.json` back to `main` with
   `[skip ci]`.

## Requirements (already encoded in the workflow)

- GitHub-hosted runner, Node ≥ 22.14, npm ≥ 11.5.1 (the workflow upgrades npm).
- Workflow permissions: `id-token: write` (npm OIDC) + `contents/issues/pull-requests: write`.
```

- [ ] **Step 2: Commit**

```bash
git add docs/RELEASING.md
git commit -m "docs: add releasing + npm OIDC bootstrap runbook"
```
